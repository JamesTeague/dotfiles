Create directory .planning/phases/00.5-audit-documentation/checks/ then write lib.sh. File MUST be sourceable (function definitions only, no top-level execution beyond env derivation). Add sentinel guard variable LIB_SH_LOADED to prevent double-load.

Required functions:
- header MSG: prints colored section header
- pass MSG: green check, increments PASS_COUNT
- fail MSG: red X, increments FAIL_COUNT; in STRICT mode also exits 1
- pending MSG: yellow dot, increments PENDING_COUNT (becomes FAIL in STRICT mode)
- assert_file PATH: file-exists; pass or pending (or fail in strict)
- assert_dir_missing PATH: pass if absent, fail if present
- assert_grep PATTERN PATH: grep -q; pass or fail (real fail — if file exists but pattern missing, it is broken)
- assert_cmd_zero_output CMD...: runs CMD, captures stdout, pass if empty / fail if non-empty (used for git ls-files filter and chezmoi diff)
- summary: prints PASS/PENDING/FAIL counts, returns 1 if FAIL_COUNT > 0

Env derivation at source time:
- REPO_ROOT from `git rev-parse --show-toplevel`
- CHEZMOI_SOURCE_ROOT from `chezmoi source-path` (empty if chezmoi missing — downstream checks become pending)
- STRICT_MODE = 1 if env var STRICT=1 OR caller passes --strict
