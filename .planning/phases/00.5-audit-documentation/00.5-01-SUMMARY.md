## Decisions Made

- **YAML validator: ruby > python3+yaml.** Mac personal has system python3 but no PyYAML. System ruby ships with YAML stdlib on macOS — always present. quick.sh tries ruby first, python3+yaml fallback, pends if neither is available. Aligned with the RESEARCH "Don't Hand-Roll" principle (use what's natively present).
- **assert_dir_missing semantics: pending by default, strict-fail in STRICT mode.** A directory targeted for wave-staged removal (e.g., flameshot, Plan 04) is "not yet done" not "broken state." Matches assert_file's pattern. Added `assert_dir_missing_strict` for true invariants (none in Phase 0.5 yet).
- **Empty `packages.yaml` placeholders recommended "keep".** Pre-noted in candidates scaffold per RESEARCH Open Q 5: the brew template ranges over them; deleting would error on missing-key without a `hasKey` guard. That guard is Phase 0 / TAX-05 template-restructure work, out of scope here.
- **full.sh has `--no-diff-gate` / `--no-quick`.** Pitfall B in RESEARCH: Mac personal has a real `.zshrc` DEBUG=1 drift RIGHT NOW that Plan 06 will reconcile. Without these flags, full.sh would fail step 3 (chezmoi diff) until Plan 06 — they let Wave 1 plans use the harness without blocking on a Wave 5 task.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking] python3+yaml unavailable on Mac personal**
- **Found during:** Task 2 (smoke-test of quick.sh)
- **Issue:** Plan specified `python3 -c yaml.safe_load(...)` for AUD-01/02 yaml validity check. Mac personal's system python3 has no `yaml` module. RESEARCH "Standard Stack" claims python3 is installed but did not verify the `yaml` module specifically. Validator would always FAIL on this machine.
- **Fix:** Re-ordered to try `ruby -ryaml` first (system ruby on macOS always ships YAML stdlib), python3+yaml as fallback. Pends rather than fails if neither is available (lack of parser != broken yaml file).
- **Files modified:** `.planning/phases/00.5-audit-documentation/checks/quick.sh`
- **Verification:** quick.sh now reports `packages.yaml is valid YAML (via ruby)` — PASS.
- **Committed in:** `ebfd942` (Task 2 commit)

**2. [Rule 1 — Bug] assert_dir_missing unconditionally failed, conflating wave-staged work with broken invariant**
- **Found during:** Task 2 (smoke-test of quick.sh; flameshot directory FAILed in default mode even though Plan 04 hasn't run yet)
- **Issue:** Original `assert_dir_missing` always called `fail` when the directory existed. But the plan uses it for `home/private_dot_config/flameshot` — a directory that PLAN 04 will remove. Same semantic class as a Wave-1+ file that doesn't exist yet (which gets `pending`, not `fail`). The asymmetry between `assert_file` (pending) and `assert_dir_missing` (fail) was a bug in the helper API.
- **Fix:** Changed `assert_dir_missing` default to `pending` (strict-fail in STRICT mode), matching `assert_file`. Added `assert_dir_missing_strict` for true invariants (no current uses in Phase 0.5).
- **Files modified:** `.planning/phases/00.5-audit-documentation/checks/lib.sh`
- **Verification:** quick.sh now reports `directory still present (not yet removed): home/private_dot_config/flameshot` as PENDING — exits 0 in default mode, fails in --strict.
- **Committed in:** `ebfd942` (folded into Task 2 commit)
